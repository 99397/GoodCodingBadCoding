
1) 코딩 스타일이란 무엇인가
Code Convention 혹은 Code Standard와 비슷하게 불린다.

때로는 성전(holy war)이 일어나곤 한다.
===== Captain America =====
function() {
    statements;
}
===========================
======== Iron man =========
function()
{
    statements;
}
===========================


2) 코딩 스타일 교육이 부족하다.
회사 안에서도 코딩 스타일에 대한 교육이 없다. 
단독 개발자가 아닌 이상 협업 체제에서는 
코드 컨벤션이라는 것이 필요한 상황이다.
비록 단독 개발환경이라 하더라도 
본인이 알아볼 수 있을 정도의 
코딩 스타일을 유지하는 것이 중요하다. 


3) 코딩 스타일을 익혀야 할 시기 
글에서는 학생때부터 시작해야 한다고 나온다.
물론 학생 때부터 할 수 있다면 좋지만 
그럴만한 환경이 아닌 이미 직장인인 우리는 어떡해야 하는가

가능한 본인의 코드와 동료의 코드를 자주 봐라.
내가 아닌 다른 사람의 스타일에서 보고 배우는 것이 좋다.
코드 리뷰나 페어 프로그래밍을 하는 것도 방법이다. 


4) 코딩 스타일을 배워야 하는 이유


5) 코딩 스타일은 수학이나 영어보다 더 중요하다.
당연한 것이다. 매일 코딩하는 사람은 당연히 중요하다.


6) 모든 언어에 코딩 스타일이 필요하다.


7) C언어로 코딩 스타일을 설명하는 이유


8) 코딩 스타일은 컴파일 방식과 하이브리드 방식에 유용하다.


9) 컴포넌트 기반 개발 방식과 코딩 스타일


10) 코더의 역량이 중요하다.


11) 코딩 관례를 문서화해 두고 통일되게 적용하라.
